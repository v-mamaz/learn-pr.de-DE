Mit einigen grundlegenden Containerkenntnissen haben Sie die Möglichkeit, Ihre eigenen Ideen zu verwirklichen. Sie können von anderen erstellte Docker-Container ausführen, Ihre eigenen Docker-Images erstellen und Ihre Arbeit mit der Community oder einfach nur Ihrem Team teilen.

Bisher haben Sie lediglich in der Sandboxumgebung die Funktionsweise von Containern untersucht. Verwenden Sie nun auch die Azure-Dienste, die es Ihnen ermöglichen, Container in der Produktion auszuführen.

Container bieten wirklich einen großen Vorteil: Sie können Code zunächst in Ihrer lokalen Umgebung entwickeln und testen, bevor Sie ihn in der Cloud bereitstellen. Sobald Ihre App oder Ihr Dienst wie erwartet in Ihrer lokalen Umgebung funktioniert, können Sie die Container in Azure ausführen. Da es sich bei einem Container um eine isolierte Umgebung für Ihre App handelt, wird er in der Cloud genauso wie lokal ausgeführt.
