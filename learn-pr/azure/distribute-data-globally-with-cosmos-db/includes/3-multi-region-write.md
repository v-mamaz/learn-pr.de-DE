## <a name="conflict-resolution"></a><span data-ttu-id="fb1b5-115">Konfliktlösung</span><span class="sxs-lookup"><span data-stu-id="fb1b5-115">Conflict resolution</span></span>

<span data-ttu-id="fb1b5-116">Mit dem Hinzufügen von Multimasterunterstützung ergibt sich die Möglichkeit, Konflikte bei Schreibvorgängen in verschiedenen Regionen zu lösen.</span><span class="sxs-lookup"><span data-stu-id="fb1b5-116">With the addition of multi-master support comes the possibility of encountering conflicts for writes to different regions.</span></span> <span data-ttu-id="fb1b5-117">Konflikte sind in der Azure Cosmos DB extrem selten und können nur auftreten, wenn ein Element gleichzeitig in mehreren Regionen geändert wird, bevor die Weitergabe zwischen den Regionen stattgefunden hat.</span><span class="sxs-lookup"><span data-stu-id="fb1b5-117">Conflicts are extremely rare in Azure Cosmos DB and can only occur when an item is simultaneously changed in multiple regions, before propagation between the regions has happened.</span></span> <span data-ttu-id="fb1b5-118">Angesichts der Geschwindigkeit, mit der die Replikation global erfolgt, sollten Sie dies gar nicht oder nur selten erleben.</span><span class="sxs-lookup"><span data-stu-id="fb1b5-118">Given the speed with which replication happens globally, you should not experience this often, if at all.</span></span> <span data-ttu-id="fb1b5-119">Azure Cosmos DB bietet jedoch Konfliktlösungsmodi, die es den Benutzern ermöglichen, zu entscheiden, wie sie mit Szenarien umgehen, in denen derselbe Datensatz gleichzeitig von verschiedenen Schreibvorgängen in zwei oder mehr Regionen aktualisiert wird.</span><span class="sxs-lookup"><span data-stu-id="fb1b5-119">However, Azure Cosmos DB does provide conflict resolution modes that allows users to decide how to handle scenarios where the same record is updated simultaneously by different writers in two or more regions.</span></span>  

<span data-ttu-id="fb1b5-120">Azure Cosmos DB bietet drei Konfliktlösungsmodi an.</span><span class="sxs-lookup"><span data-stu-id="fb1b5-120">There are three conflict resolution modes offered by Azure Cosmos DB.</span></span> 
* <span data-ttu-id="fb1b5-121">**Last-Writer-Wins (LWW)**: Konflikte werden basierend auf dem Wert einer benutzerdefinierten ganzzahligen Eigenschaft im Dokument gelöst.</span><span class="sxs-lookup"><span data-stu-id="fb1b5-121">**Last-Writer-Wins (LWW)**, in which conflicts are resolved based on the value of a user-defined integer property in the document.</span></span> <span data-ttu-id="fb1b5-122">Standardmäßig wird „_ts“ verwendet, um das zuletzt geschriebene Dokument zu bestimmen.</span><span class="sxs-lookup"><span data-stu-id="fb1b5-122">By default _ts is used to determine the last written document.</span></span> <span data-ttu-id="fb1b5-123">Last-Writer-Wins ist der Standardmechanismus für die Konfliktbehandlung.</span><span class="sxs-lookup"><span data-stu-id="fb1b5-123">Last-Writer-Wins is the default conflict handling mechanism.</span></span>
